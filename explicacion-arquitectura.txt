 GUÍA DE ORGANIZACIÓN DEL PROYECTO - SIGLOR - EDA2 (2026)
===========================================================================

Muchachos, armé esta estructura para que no nos choquemos los unos con los 
otros mientras programamos y para que el repositorio de GitHub quede bien 
prolijo para la entrega. Acá les explico qué va en cada lado:

---------------------------------------------------------------------------
1. ¿DÓNDE METO MIS ARCHIVOS?
---------------------------------------------------------------------------

/cabeceras: Acá van solo los archivos .h. Es donde definimos "qué" cosas 
va a tener el programa (por ejemplo, qué datos lleva una Ciudad o una Ruta). 

/fuentes: Acá van los archivos .cpp. Es el código real, donde escribimos 
la lógica de los algoritmos y las funciones. Si el programa hace algo, 
el código está acá adentro.

/datos: Acá se guardan los archivos .dat (los binarios). No borren lo que 
hay acá porque es la "memoria" del programa; si lo borran, hay que cargar 
todas las ciudades de nuevo.

/recursos: Acá tiramos las fotos, el mapa de fondo y cualquier cosa visual 
que necesite el programa para no verse vacío.

/compilacion y /ejecutables: Estas carpetas son para los archivos que crea 
la compu cuando compilamos. No hace falta que toquen nada acá, Git las va 
a ignorar para no subir basura innecesaria al servidor.

---------------------------------------------------------------------------
/compilacion y /ejecutables: Git las va a ignorar. (Si no las ven en el
repo es por eso, se les van a crear solas cuando compilen en su PC).
